Default Format Used by Everest

The default formats used by Everest for request and answer files are given below:

Transaction Request

If a request format is not specified for a processor, Everest generates requests in the following format:

SALE = , CLERK, ORDERNO, ACCOUNT, EXP + SWIPEINFO,'AMOUNT, ZIP, ADDRESS,, TAX,,,,,, ORDERNO,,, CVV2

SHIP = , CLERK, ORDERNO, ACCOUNT, EXP + SWIPEINFO, AMOUNT, ZIP, ADDRESS

REFUND = , CLERK, ORDERNO, ACCOUNT, EXP + SWIPEINFO, AMOUNT

BOOK = , CLERK, ORDERNO, ACCOUNT, EXP + SWIPEINFO, AMOUNT, ZIP, ADDRESS,, TAX,,,,,, ORDERNO,,, CVV2

FORCE = , CLERK, ORDERNO, ACCOUNT, EXP +SWIPEINFO, AMOUNT, APPROVECODE, ZIP, ADDRESS

AUTH = , CLERK, ORDERNO, ACCOUNT, EXP + SWIPEINFO, AMOUNT, ZIP, ADDRESS,, TAX,,,,,, ORDERNO,,, CVV2

Answer

If a request is not specified for a processor, Everest translates the response assuming that the answer file is in the format given below:

APPROVE=2,1

RESPONSE=3,6

AVS=9,1

CVV2=10,1

TRANID=12,12
