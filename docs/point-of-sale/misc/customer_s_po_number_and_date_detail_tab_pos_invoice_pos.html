Customer’s PO Number and Date

The Customer’s PO No. field displays the number of the customer’s purchase order that links the sales order with the customer’s order. This helps you track the sales order to the copy of the customer’s order. If a PO number has been duplicated for a customer, Everest displays the Confirm window. Click Yes to confirm that you want to use the same PO number again. Click No to enter another PO number.

You can set a preference by selecting the option Prompt for duplicate PO number from the same customer (path: File > Setup > Preferences > Flow Control > Documents > Prompts > Sales Document > Sales Document 2 tab) to display a message if the PO number entered (in the new sales document) has been already been entered in another sales order for the same customer. This helps to avoid duplication of sales documents for a customer.

Enter the number of the customer’s purchase order in the Date field.
