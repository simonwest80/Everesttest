Set Barcode Preferences (Optional)

You can set the default Barcode Type for items in your flow control preferences. This is an optional procedure. If you select a barcode type other than “None”, you will be presented with a confirmation message if the item code does not match the selected standard.

Figure: Barcode Confirmation Message

To set up your barcode type preference, do the following:

  • From the Everest main menu, select File > Setup > Preferences > Flow Control > Items.
  • On the Item details tab, select your barcode type from the Bar code type field drop-down list. Your choices are None, UPC, C39, C128, and I25.  This will be the default barcode standard to be used for barcodes for new items. The purpose of this setting is to force you to create new items that match the standard selected. If you put the barcode in the Model field or a field other than the Item Code field, it is best to set the Bar code type to “None”.

If you select UPC as the Bar Code Type, the items created in Everest have to meet the 12 digit UPC standards for barcodes.

Figure: The Items Dialog Box – Item Details Tab showing Bar Code Type Preference

  • Click OK.
