# Common Date Field Options


The options common to all the fields in the **Dates** tab of the **Find Sales Orders** dialog  box are enumerated below. It also elucidates the result of the search  displayed for each of the options.


| Option | Search Result |
| All | All documents created (irrespective of their creation  date). |
| Today | Documents created on the login  date.<br/><br/><br/>![]({{site.sp_baseurl}}/img/lens.gif) [Login  Time and Date]({{site.wwe_chm}}/everest-client/logging-in/time_zone.html) |
| This week | Documents created from the beginning of the current  week to the end of current week. **Everest**  considers that a week spans from Monday to Sunday. |
| This week-to-date | Documents created from the beginning of the current  week to the current date. This option is useful if you have logged in  using a date prior to the current date.<br/><br/><br/>![]({{site.sp_baseurl}}/img/example.gif) If  the current date is  Jan.  14**th**, 2009, and you login  with the date as  Jan.9**th**, 2009, all sales orders created  from Monday - Jan. 5**th**  to  Jan.9**th** are displayed. |
| This month | Documents created from the beginning of the current  month to the end of current month. |
| This month-to-date | Documents created from the beginning of the current  month to the current date. |
| This year | Documents created from the beginning of current year  to end of the current year. |
| This year-to-date | Documents created from the beginning of current year  to the current date. |
| Yesterday | Documents created on the date prior to the login  date. |
| Last week | Documents created during the week preceding the week  pertaining to the login  date. |
| Last week-to-date | Documents created from the beginning of the last week  to the current date. |
| Last Month | Documents created during the last month. The last month  is the month preceding the login  date's month. |
| Last month-to-date | Documents created from the beginning of the last month  to the current date. |
| This fiscal quarter | A fiscal quarter comprises three months. Documents created  from the beginning of the current quarter to the end of the current quarter. |
| This fiscal quarter-to-date | Documents created from the beginning of the current  fiscal year to the current date. |
| This fiscal year | Documents created from the beginning of the current  fiscal year to the end of the current of fiscal year. |
| This fiscal year-to-date | Documents created from the beginning of the current  fiscal year to the current date. |
| Last fiscal quarter | Documents created from the last fiscal quarter to the  end of the current quarter. |
| Last fiscal quarter-to-date | Documents created from the beginning of the last fiscal  year to the current date. |
| Last fiscal year | Documents created from the beginning of the last fiscal  year to the end of current of fiscal year. |
| Last fiscal year-to- date | Documents created from the beginning of the last fiscal  year to the current date. |
| Next fiscal quarter | Documents created as on the corresponding fiscal quarter  of the next year.<br/><br/><br/>![]({{site.sp_baseurl}}/img/example.gif) You have logged in the previous fiscal quarter (April to  June 2008), but you need the list of customers for the current fiscal  quarter (April to June 2009), you can select this option. |
| Next Fiscal Year | Documents created from the beginning to the end of next  fiscal year.<br/><br/><br/>![]({{site.sp_baseurl}}/img/example.gif) You have logged in the previous fiscal year (January to December  2008), but you need the list of customers for the current fiscal year  (January to December 2009), you can select this option. |
| Custom | Documents created on the dates or the date range specified  by you. |
| Ask | This field allows you to make a date field mandatory. If the filter  parameters are saved, the saved filter cannot be used again, unless a  value is specified for the field that has the **Ask** option selected.<br/><br/><br/>![]({{site.sp_baseurl}}/img/example.gif) You  create a find filter for sales orders and select the **Ask** option in the **Due Date** field.  The next time, you will not be able to use the saved filter, unless you  specify a value in the **Due Date** field. |
