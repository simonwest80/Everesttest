Manual Sales Orders

Manual sales orders are created when you create an entry in a Receipts Journal with the entry type ‘Miscellaneous Deposit’. The document number is auto-generated prefixed with ‘M’ to denote that it is a manual sales order. The sequence number for manual sales orders specified in the login location or sub-location is used. If the location’s sequence number is blank, the sequence number for the default location is used.

You can make modifications from within the manual sales order profile and convert sales orders to sales invoices in the same manner as system documents.

Viewing Manual Sales Orders

You can view manual sales orders from the Sales Orders browser by selecting the Accounting option from the View menu (path: Sales Order browser > View > Accounting). Manual Sales Orders have a SO# that is prefixed with “M”.  Manual documents also display “Accounting” in the Type column.

Modify Manual Sales Orders

You can modify manual sales orders from the Sales Order profile just as you would with system documents. This is the preferred method for making manual sales order modifications since only limited modifications are possible from the receipts journal.

To modify manual sales orders, do the following:

  • From the Sales Orders browser, select View > Accounting.
  • Select the manual sales order (The SO# begins with “M” and the Type column displays “Accounting”) and press the Enter key or double-click it. You will see the manual sales order profile.
  • Make the necessary modifications. You can update any of the following items:
  1. Add or edit Document Notes (Click the icon next to the Document #, press **Shift
    • F6, or select **Options > Document > Document Notes.)
    • Apply refunds and view the Payment History (Select Options > Payments.)
    • Update custom fields (Select Options > Document > Custom fields or press Ctl + Alt + C.)
    • Change document alias (Click the icon next to the Document # or press **Ctl
    • Alt +** D.
    • Update fields in the manual sales order, including Contact, Ship To, Sales Rep, Terms, Due date, Override, Customer’s PO Number and Date, Shipment Tracking #, Reference, and Loc/Sub-loc (Location/Sub-Location).
