Data Dictionary

Use the Data Dictionary report to view the structure of tables and indexes in Everest. The report is useful when customizing Everest reports for specific user requirements. It is also useful in the creation of reports, as it enables the user to understand how the tables within Everest can be linked. This report displays the following details:

  • Table details, such as:

  • table name
  • table description
  • indexes
  • number of field types in the table

  • number of data types in the table

  • Field details in each table, such as:

  • field name
  • field description
  • data type
  • size of field, and
  • number of decimal places, if the field is a numeric type

A table is a data structure used to organize information. In a relational database, as in Everest, a table organizes the information about a single topic into rows and columns. Each single piece of data is a field in the table. A column consists of all the entries in a single field. Fields, in turn, are organized as records, which are complete sets of information, each of which comprises a row.

To generate the Data Dictionary report, do the following:

  • From the main menu bar, select Reports > Miscellaneous > Data Dictionary. You will see the Data Dictionary Report dialog box.

  • Specify the report parameters and click:

  • Preview, to preview the report in the Report Preview dialog box
  • Print, to print the report
  • Export, to save the report in text, Excel, HTML or Crystal Report file formats
