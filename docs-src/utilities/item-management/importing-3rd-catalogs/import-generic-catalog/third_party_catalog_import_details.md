# Third Party Catalog Import - Import Details


This section is used to specify the type of import process that needs  to be carried out. **Everest** allows  you to create a new catalog, overwrite an existing catalog or update prices  and stock quantities in an existing catalog.


Import Type


**Create new catalog**
: Select this option to create a new third party catalog.


**Overwrite existing catalog**
: Select this option if you want to overwrite an existing  third party catalog. This will delete all existing items, brands, categories  and manufacturers and will recreate them.


**Update prices and stock quantities in existing  catalog**
: Select this option to update the third party catalog  item prices and stock quantities existing in your database. When you select  this option, the following are enabled:

- **Import 
 new items and related information** - Select this check box if you  want to update the prices and stock quantities and also import items that  are not present in your existing database.
- **Discontinue 
 items which no longer exists in catalog** - Select this check box  to update the prices and stock quantities and also discontinue those items  which are present in your existing database, but no longer exist in the  third party catalog.



**Third Party Catalog**
: Select the Third Party Catalog code that is already  defined in **Everest**. The drop-down  list displays all the active third party catalogs.


{:.note}
This field is disabled if you select the **Create new catalog** option.


If you select the **Create 
 new catalog** option, **Items**,  **Categories**, **Brands**  and **Manufacturers** check boxes  are displayed. These options are used to specify the third party catalog  data that needs to be imported. Depending on the option selected, the  respective data is imported.


{:.note}
If you do not select the **Categories**  option, category records are automatically created for each new category  defined for an item.
: Click **Next**.  You will see the **Catalog 
 Details** section.
