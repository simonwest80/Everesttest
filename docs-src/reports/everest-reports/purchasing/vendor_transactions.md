# Vendor Transactions


The **Vendor Transactions** report  lists all business transactions made with vendors during the specified  time period. The history of all the transactions can be analyzed to determine  vendors with whom business is profitable. The report displays vendor balances  for all purchase and payment journals irrespective of whether they have  been posted or not.


The report displays the following:

- Account code and  name of the vendor.
- Address details  of the vendor including telephone and fax numbers, if specified in the  **Vendor** profile.
- Purchase and purchase  return documents created for the vendor, along with document date, number  and amount.
- Whether the vendor  has put you on credit hold or not.
- Current net balance  of the vendor.
- The payment term  as specified in the **Accounting**  tab of the **Vendor** profile.
- Reference number  and reference date for check payments and refunds.



{:.note}
Employees created as vendors are not included  in this report.


{:.steps}
To generate the Vendor Transactions report, do the following:

- From the main  menu bar, select **Reports &gt; Purchasing 
 &gt; Vendor Transactions**. You will see the **** [**Vendor 
 Transactions** dialog box]({{site.rpt_baseurl}}/misc/vendor_transactions_dialog_box_rep.html).
- Specify the  report parameters and click:


- **Preview**,  to preview the report in the **Report Preview** dialog box, before printing
- **Print**,  to print the report
- **Export**,  to save the report in text, Excel, HTML or Crystal Report file formats
