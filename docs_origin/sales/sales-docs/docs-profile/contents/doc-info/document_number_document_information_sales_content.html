Document Number

The Document # field displays the number assigned to a sales document when it is created. Sequence numbers for sales documents are set in the Location/Sub-Location profile (path: File > Setup > Accounting > Location/Sub-Location > Options > Edit > In NewWindow > Invoicing tab), of the logged in location/sub-location from which you create the sales document.

Location and Sub-Location

If  sequence numbers are not defined for the login location or sub-location,  the sequence number specified in the Default Location field (path: File > Setup > Preferences > Defaults > Accounting > Miscellaneous tab) is assigned to the document.

Default Location

Everest automatically assigns the consecutive number to the sales document created subsequently.

If you change the location sequence number, you cannot assign a number that is already used. The system prompts you to set a proper range of sequence numbers.

If a quote is converted to an order or an order to an invoice, and the login location is changed before saving the document, the sequence number set for the logged in location at the time of creating the order or invoice (although this is different from the location for the original quote or order) is used.

Numbering for Document Aliases

Sequence numbers are assigned continuously. Documents using a different alias are not assigned a different sequence.

If you create a sales quote with the number 1000 and the next sales quote with an alias ‘Service Quote’, the document number assigned to the ‘Service Quote’ would be 1001.

Manual Numbering

You cannot number the documents manually in Everest. You can however, change the sequence number by altering it from the Location profile (path: File > Setup > Accounting > Location/ Sub-Location > Options > Edit > In NewWindow > Invoicing tab). Every time you change a sequence number, Everest allots the next number in sequence to the documents, thereafter.

Sequence numbers once assigned are not re-assigned by Everest even if the document is discontinued or purged.
