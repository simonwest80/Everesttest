Transactions

Transaction #
Transaction # is the unique number used to identify the transaction. You can have one or more transactions per journal. You can navigate between transactions in the same journal by clicking on the down arrow and selecting the required transaction number. The transaction number is created automatically by Everest.
(Transaction Narration)
Click this icon to view the Transaction Narration dialog box. Use this dialog box to enter notes for each transaction in the journal. Notes are not automatically generated for system journals nor can they be appended manually.
Date
This field displays the date on which the sales document was created.
Sales Representative
The sales representative who created the document is displayed in this field.
Customer
This field displays the name and billing address of the customer for whom the document was created. The Customer field cannot be edited for a system journal.
On Credit Hold and Credit Limit
These fields display whether the customer is on credit hold and the customer’s credit limit. The current credit limit and hold status is displayed and not the limit at the time of creating the document.
These details are automatically displayed from the Customer profile (path: Invoicing > Customers > Options > Edit > In New Window > Payment Information tab).
The credit limit is displayed in the customer’s currency.
Source
This option displays the source type describing the classification of this transaction. The source type for a Sales Invoice is INVand source type of a Credit Memo is CRM. The source type cannot be modified for a system journal.
Source Types
Currency
The document’s currency code is displayed in this field.
Exchange Rate
The exchange rate for the document is displayed in this field. The exchange rate is the conversion rate of the foreign currency to your base currency.
This is a display field and cannot be modified for system journals.
Foreign Currency Information - Sales Documents
Document Type
Accounting entries are generated only for final documents (i.e. Sales Invoices and Credit Memos). The document type displayed is thus either Sales Invoice or Credit Memo.
Document Type Description
Sales Invoice Used when your company sells goods or provided services to a customer.
Credit Memo Used when your company provides the customer an allowance for return of goods or services.
Document #
The document number of the Sales Invoice or Credit Memo is displayed in this field. This number uniquely identifies the document in Everest. The document number is automatically generated when the document is created.
Customer’s PO # and date
These fields display the customer’s purchase order number and date, if any, specified on the sales invoice or credit memo.
Terms
The payment terms for the document is displayed in this field.
Payment Terms - Sales Documents
Due date
This field displays the due date by which the customer has to pay the dues on the document. This date is computed based on the payment terms specified for the document.
Jurisdiction
This field identifies the jurisdiction for the sale. Sales taxes are calculated for the taxes applicable in the jurisdiction.
