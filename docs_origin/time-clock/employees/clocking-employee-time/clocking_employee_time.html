Clocking Employee Time

Time clocking is used to maintain a record of the number of hours an employee works per day.

The system automatically clocks the start time (of the working hours), when an employee’s name is entered. Only if the clock in time is validated, can the employee be clocked out for the day.

When the employee name is re-selected the system automatically clocks the end time (of the working hours), which is then validated. Validation completes the clock out process. The employee cannot clock in again if the validation process is not completed.

Clocking employee time keeps track of the following such as:

  • If employees are on time.
  • If employees adhere to work shifts.
  • The actual number of hours worked in the office premises.
  • The number of hours employees were out on office work.

The time zone and daylight savings time details default from the client workstation’s operating system settings and not the Everest settings when recording a transaction. Once a transaction is saved, it does not change even when viewed from different time zones or with different daylight savings settings.
